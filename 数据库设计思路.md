## 🎯 一、整体架构设计思路

### 1.1 设计原则

text

```
✅ 模块化设计：将业务功能拆分为独立模块
✅ 继承统一：所有表继承BaseEntity，统一审计字段
✅ 动静分离：高频变更表与静态信息表分离
✅ 父子关系：支持分类层级和树形结构
✅ 数据驱动：所有展示内容后台可配置
```



### 1.2 三大层次架构

text

```
┌─────────────────────────────────────┐
│     展示层：前端页面展示            │
├─────────────────────────────────────┤
│     业务层：19张核心业务表          │
├─────────────────────────────────────┤
│     基础层：统一BaseEntity + 索引    │
└─────────────────────────────────────┘
```

## 📋 二、19张表详细架构说明

### 2.1 公司核心模块（5张表）- **静态低频变更**

**特点**：初始化后基本不变，适合缓存

| 序号 | 表名                 | 功能         | 关联关系           | 数据示例                 |
| :--- | :------------------- | :----------- | :----------------- | :----------------------- |
| 1    | **`company_info`**   | 公司基本信息 | 无父表，核心配置表 | 公司名称、LOGO、文化理念 |
| 2    | **`company_metric`** | 实力指标     | 无父表，展示用     | "项目总投资80亿+"        |
| 3    | **`core_value`**     | 核心价值     | 无父表，首页展示   | "创新引领"、"品质卓越"   |
| 4    | **`certification`**  | 企业认证     | 无父表，资质展示   | IATF16949认证            |
| 5    | **`contact_info`**   | 联系信息     | 多地点，可扩展     | 总部、分公司联系方式     |

**构思**：这5张表构成公司"名片"，一次配置长期使用，支持中英文，SEO友好。

### 2.2 产品业务模块（4张表）- **父子树形结构**

**特点**：层级分类 + 产品详情分离

text

```
product_category (父表)
    ├── 智能触控领域 (一级分类)
    │   ├── G1系列 (二级分类)
    │   └── 教育触控 (二级分类)
    └── 车载显示领域 (一级分类)
        └── 中控大屏 (二级分类)
            └── product (关联产品表)
                └── product_spec (产品规格子表)
```



| 序号 | 表名                   | 功能     | 父子关系          | 关键字段           |
| :--- | :--------------------- | :------- | :---------------- | :----------------- |
| 6    | **`business_area`**    | 业务领域 | 一级分类          | 智能触控、车载显示 |
| 7    | **`product_category`** | 产品分类 | `parent_id`自关联 | 树形分类结构       |
| 8    | **`product`**          | 产品主表 | 关联category表    | 产品详情、应用场景 |
| 9    | **`product_spec`**     | 产品规格 | `product_id`关联  | 技术参数、规格明细 |

**构思**：四表联动形成完整产品体系，支持无限级分类，规格参数独立存储便于维护。

### 2.3 内容资讯模块（4张表）- **标签化内容管理**

**特点**：分类+标签双重管理，内容灵活

text

```
article_category (内容分类树)
    ├── 新闻中心 (一级)
    │   ├── 公司新闻 (二级) ──┐
    │   ├── 行业动态 (二级)   ├── news (新闻主表)
    │   └── 党建动态 (二级) ──┘
    └── 技术文章 (一级)
        └── article_tag (标签库)
            └── news_tag_relation (关联表)
```



| 序号 | 表名                    | 功能     | 关联关系          | 特殊设计               |
| :--- | :---------------------- | :------- | :---------------- | :--------------------- |
| 10   | **`news`**              | 新闻资讯 | 关联category表    | 支持党建、招标特殊字段 |
| 11   | **`article_category`**  | 文章分类 | `parent_id`自关联 | 多级分类树             |
| 12   | **`article_tag`**       | 文章标签 | 独立标签库        | 颜色、类型分类         |
| 13   | **`news_tag_relation`** | 标签关联 | 多对多中间表      | 解耦设计               |

**构思**：分类负责结构化组织，标签负责灵活检索，中间表实现多对多关系。

### 2.4 技术研发模块（2张表）- **技术实力展示**

**特点**：展示公司硬实力

| 序号 | 表名                  | 功能     | 关联关系          | 核心数据             |
| :--- | :-------------------- | :------- | :---------------- | :------------------- |
| 14   | **`rd_team`**         | 研发团队 | 独立展示          | 团队介绍、研发投入8% |
| 15   | **`core_technology`** | 核心技术 | `parent_id`技术树 | 微纳材料、智能工艺   |

**构思**：独立模块突出技术实力，`parent_id`支持技术领域树形分类。

### 2.5 人力资源模块（2张表）- **组织+招聘**

**特点**：组织架构树 + 岗位管理

text

```
department (组织架构树)
    ├── 董事会 (level=1)
    ├── 总经理 (level=2)
    ├── 研发中心 (level=3)
    ├── 制造中心 (level=3)
    └── 营销中心 (level=3)
        └── job_position (招聘岗位)
```



| 序号 | 表名               | 功能     | 父子关系            | 特殊字段            |
| :--- | :----------------- | :------- | :------------------ | :------------------ |
| 16   | **`job_position`** | 招聘职位 | 关联department表    | 结构化职责要求      |
| 17   | **`department`**   | 组织架构 | `parent_id`树形结构 | `dept_path`快速查询 |

**构思**：两张表覆盖从组织架构到具体岗位的全流程，`dept_path`字段优化树查询性能。

### 2.6 关于我们模块（3张表）- **公司历程展示**

**特点**：时间线 + 基地 + 合作伙伴

| 序号   | 表名                   | 功能     | 展示形式   | 数据特点             |
| :----- | :--------------------- | :------- | :--------- | :------------------- |
| 18     | **`company_timeline`** | 发展历程 | 时间轴展示 | 年份、事件、图片     |
| 19     | **`base_showcase`**    | 基地展示 | 地图+图片  | 位置、面积、产能     |
| (已有) | **`partner`**          | 合作伙伴 | Logo墙展示 | 类型、行业、合作时间 |

**构思**：三张表分别从时间、空间、合作三个维度展示公司发展全貌。

## 🔗 三、关键关联关系设计

### 3.1 父子关系设计（树形结构）

sql

```
-- 4张支持树形结构的表
1. product_category (产品分类树)
   parent_id -> 自身id，支持无限级分类
   
2. article_category (内容分类树) 
   parent_id -> 自身id，新闻/文章分类
   
3. department (组织架构树)
   parent_id + dept_path，快速定位层级
   
4. core_technology (技术领域树)
   parent_id，技术领域分类
```



### 3.2 主从关系设计

sql

```
-- 3组主从表设计
1. product (主表) ← product_spec (从表)
   产品基本信息 + 详细规格分离
    
2. news (主表) ← news_tag_relation (从表)
   新闻内容 + 标签关联解耦
    
3. department (主表) ← job_position (从表)
   部门信息 + 招聘岗位关联
```



### 3.3 多对多关系设计

sql

```
-- 标签系统的优雅设计
article_tag (标签库，独立维护)
    ↑
news_tag_relation (关联表，解耦设计)
    ↑
news (新闻内容，不直接存储标签)
```

## 🏗️ 四、技术亮点与创新

### 4.1 统一基础架构

java

```
// 所有19张表都继承BaseEntity
public class BaseEntity {
    private Long id;                    // 主键
    private LocalDateTime createdAt;    // 创建时间
    private LocalDateTime updatedAt;    // 更新时间
    private Long createdBy;            // 创建人
    private Long updatedBy;            // 更新人
    private Long deptId;               // 部门ID（数据权限）
    private Byte deleted;              // 逻辑删除
}
```



### 4.2 JSON字段灵活应用

sql

```
-- 6张表使用JSON字段存储动态数据
1. product.images           -- 产品图片数组
2. product.application_scenes -- 应用场景
3. news.party_participants -- 参与人员
4. job_position.job_responsibilities -- 岗位职责
5. base_showcase.facilities -- 基地设施
6. rd_team.research_areas  -- 研究领域
```



### 4.3 性能优化设计

sql

```
-- 关键索引设计
1. 分类表：parent_id + order_index 联合索引
2. 新闻表：category + publish_date 联合索引  
3. 产品表：category_id + status + order_index
4. 组织表：parent_id + dept_path 快速查询
```
